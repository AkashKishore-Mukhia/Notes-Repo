# Docker
Docker is an OS‑level virtualization (or containerization) platform, 
which allows applications to share the host OS kernel instead of running a 
separate guest OS like in traditional virtualization. This design makes Docker 
containers lightweight, fast, and portable, while keeping them isolated from 
one another.


**What is a container ?**
- A way to package application with all the necessary dependencies and configuration.
- Portable artifact easily shared and move around.
- Makes the dev and dep efficient.

**Where do container live?**
- Container lives in container repository.
- Private and Public(docker-hub) repositories.

Before container development lifecycle:
- Each Developer need to install dependency and configuration in the system.
- Installation process is different on each OS.

After Container development lifecycle:
- In container dependencies and configurations are packaged into a single isolated environment.

**What is a dockerfile?** 
A dockerfile is a text file that contains a set of instructions used to build a docker image.
