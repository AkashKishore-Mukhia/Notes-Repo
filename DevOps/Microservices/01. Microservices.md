# Microservices

Breaking down an Application into multiple different services which are developed and deployed
independently.

**What is Monolithic architecture?**
- All Component are part of a single unit.
  - e.g. shopping app(usr-auth, shopping cart, payment, notification) all as a single unit.
- Everything is developed deployed and scaled as one single unit.
- App must be written with 1 tech stack
- Teams must be careful to not affect each other's work as it's a single whole app.
- 1 single artifact, so we must redeploy the entire application on each update


## Microservices:
- Application must split based on business functionality.
- one microservice should provide only one service.
- should be loosely coupled with each-other(microservice).

**How will these microservices communicate with each other?**
1. Communication via API calls. -> synchronous
2. Communication via Message brokers(msg queue) -> asynchronous
3. Communication via service mesh
