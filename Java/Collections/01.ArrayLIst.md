## 1. ArrayList vs. LinkedList

While both classes implement the `List` interface and share similarities (duplicates allowed, insertion order preserved), they have different internal structures and performance characteristics.

| Feature | ArrayList | LinkedList |
| :--- | :--- | :--- |
| **Internal Data Structure** | Resizable/Growable Array | Double Linked List |
| **Memory Allocation** | Consecutive memory locations | Non-consecutive memory locations |
| **Best Use Case** | **Retrieval operations** (due to `RandomAccess` interface) | **Insertion/Deletion in the middle** |
| **Worst Use Case** | Insertion/Deletion in the middle (requires internal **shift operations**) | Retrieval operations (complex to find elements non-consecutively) |

## 2. The Vector Class
`Vector` is a legacy class (JDK 1.0) that acts as a synchronized, thread-safe version of a list [8-10].

### Key Properties
*   **Thread Safety:** Every method is **synchronized**, making the object thread-safe [9].
*   **Interfaces:** Implements `Serializable`, `Clonable`, and `RandomAccess` [9].
*   **Capacity Growth:** When a vector reaches its maximum capacity, it creates a new object with **double** the capacity (New = Current * 2) [11, 12].

### Constructors
1.  `Vector v = new Vector();` — Initial capacity of **10** [12, 13].
2.  `Vector v = new Vector(int initialCapacity);` — Specific starting size [14].
3.  `Vector v = new Vector(int initialCapacity, int incrementalCapacity);` — Allows you to define exactly how much the capacity grows [15].
4.  `Vector v = new Vector(Collection c);` — Converts any collection into an equivalent Vector [16].

### Legacy Method Names
Because it is from an older "generation," Vector methods have more descriptive, lengthier names [17, 18].
*   **Adding:** `addElement(Object o)` [18, 19].
*   **Removing:** `removeElement(Object o)`, `removeElementAt(int index)`, `removeAllElements()` [18, 19].
*   **Retrieval:** `elementAt(int index)`, `firstElement()`, `lastElement()` [20].

## 3. The Stack Class
`Stack` is a child class of `Vector` specifically designed for **Last-In-First-Out (LIFO)** operations [21, 22].

### Specific Methods
*   `push(Object o)`: Inserts an object onto the top [23, 24].
*   `pop()`: Removes and returns the top object [23, 24].
*   `peek()`: Returns the top object without removing it [23, 24].
*   `empty()`: Returns boolean if the stack is empty [24, 25].
*   `search(Object o)`: Returns the **offset** of the element (position from the top starting at 1). Returns -1 if not found [24, 26].

**Example of Offset vs. Index:**
```java
Stack s = new Stack();
s.push("A"); // Index 0, Offset 3
s.push("B"); // Index 1, Offset 2
s.push("C"); // Index 2, Offset 1 (Top)

System.out.println(s.search("A")); // Output: 3
System.out.println(s.search("Z")); // Output: -1

4. Cursors: Enumeration
A cursor is used to retrieve objects from a collection one by one
. Enumeration is the legacy cursor used primarily with legacy classes like Vector
.
Methods
hasMoreElements(): Checks if more elements are available
.
nextElement(): Returns the next object
.
Example: Filtering Even Numbers
Vector v = new Vector();
for (int i = 0; i <= 10; i++) {
    v.addElement(i);
}

Enumeration e = v.elements(); // Create cursor
while (e.hasMoreElements()) {
    Integer i = (Integer) e.nextElement();
    if (i % 2 == 0) {
        System.out.println(i); // Prints: 0, 2, 4, 6, 8, 10
    }
}
