# Static Modifier
- We can't access instance member directly from static area, but we can access from instance area directly.
- We can access static memebers both from instance and static are directly.
Note: Overloading concept applicable including `main` method but JVM can always call string array argument main method only.
```java
class Test {
    public static void main(String[] args) {
        System.out.println("String[] args method");
    }
    public static void main(int[] args) {
        System.out.println("String[] args method");
    }
}
```
- Inheritance concept applicable for static method including `main` method.
- It seems overiding applicable for static methods, but it's not overriding but method hidding.
```java
class p {
    public static void main(String[] args) {
        System.out.println("Parent main method");
    }
}

class c extends p {
    public static void main(String[] args) {
        System.out.println("child main method");
    }
}
```
- inside method implementation if we are using atleast one instance variable, then that method talks about a particular object, hence we should declare method as instance method.
- inside method implementation if we are not using any instance variable, then this method no way related to a particular object, hence we have to declare such type of method as static method. irespective of we are using static variable or not.
- For static method implementation should be available.
