# String


| String                       | StringBuffer             |
|------------------------------|--------------------------|
|String is immutable in nature | StringBuffer is muttable |


## Mutability vs Immutability

```java
String s = new String("Akash");
s.concat("Kishore");
System.out.println(s);

// output: Akash    
```
> note: since string objects are muttable in nature, `s.concat` create a new object containing "Kishore". And
as it doesn't have any reference variable it gets garbage collected.


```java
String s = new StirngBuffer("Akash");
s.append("Kishore");
System.out.println(s);

// output: AkashKishore
```

## "==" vs .equals methods

**== :** is always used for reference comparison

**.equals():** is defined it in `object`(parent) class for reference comparison but in `string`(child) class it is overridden 
for content comparison.


## Heap and String constant pool(SCP)
- In java onward 1.6v the heap are divided into `heap` and `scp` memory.
- whenever a `new` keyword used to create object, compulsory one object will be created in the heap area.
- Literals objects are stored(scp area) here and if a literal object is already present, jvm skips creating new object
in the `scp` area.


> !note: if one or more literal are constant then at compile time the operation are performed if any operation present not at runtime. \
> if used final keyword at the time of variable declaration, then variable used anywhere will be updated
at compile time. \
``
final string s4 = "you cant";
``
