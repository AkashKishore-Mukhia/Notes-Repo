# Domain Name System

DNS translate domain name into ip address to locate a server hosting a service.

## Components of DNS:
1. Name Space
2. Name Registry
3. Name Resolution

### Name Space:
- Domain are interpreted by DNS from right to left

```
[www].[google].[com][.]
{.....................} FQDN(Fully Qualified Domain Name) <= 255 character
[third-level-domain][second-level-domain][TLD][root]
{..............labels........................}{label of null}
```
**FQDN vs URL:** \
FQDN = domain-name \
URL = protocol + domain-name + path

> note: each label can be 63 char long, allowed chars are {a-z, 0-9, '-'}
![how does DNS resolve IP](https://assets.bytebytego.com/diagrams/0176-dns-look-up.png)

### Name Resolution:

**Local name resolution:**
- Resolved domain name in the local machine without the use of an external name servers.
- Local name resolution happens using a text file know as host table
  - `windows: C:\Windows\System32\drivers\etc`
  - `linux: \etc\`

### DNS Resolver:
- It's a server which resolve the domain name into ip address communicating back and forth with name servers.
- DNS have the info about the root name server, which is store in root hints file in DNS resolver.

### DNS Resolution Types:
1. Iterative Resolution (server send referral servers info to client)
2. Recursive Resolution (server responsibility to resolve the domain name)
